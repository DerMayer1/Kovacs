# Kovacs V0.3.0 Charter

Status: normative

## Objective

Turn the V0.2 observer into a Daily Staff Engineer Operating System. Kovacs must connect daily execution to a confirmed 90-day mission and rolling weekly outcome, generate reviewable plans, guide the active checkpoint, retain only structured learner evidence, and show whether competence is becoming reliable through practice.

The durable main goal remains:

> Become an Elite AI Systems Staff Engineer, using OpenAI as the benchmark of engineering efficiency, judgment, and impact.

## Product contract

- Goal hierarchy is `main goal -> 90-day mission -> rolling week -> daily objective -> checkpoint`.
- Kovacs drafts plans; the user confirms setup, weekly outcomes, and daily plans before they become active.
- Low-risk checkpoint progression is automatic after the user records evidence.
- A successful day requires a concrete output, validation, strategic alignment, and a retained lesson.
- Competence begins `unverified` and advances only through sourced outcomes, not titles, hours, screen activity, or self-rating.
- Raw screenshots and captured content remain ephemeral. Durable state contains structured plans, evidence, memories, and invocation telemetry only.
- Every durable memory is visible, confirmable, pinnable, and deletable.
- Kovacs remains advisory. It cannot click, type, send, submit, publish, commit, apply to jobs, or act after a meeting.
- Codex CLI remains the only model gateway. Local deterministic logic handles state, cooldowns, duplicate suppression, evidence calculations, and memory controls.

## Included

- first-run calibration and schema-constrained 90-day mission proposal;
- explicit rolling-week proposal and confirmation;
- objective-quality challenge and daily-plan confirmation;
- two to six evidence-bearing checkpoints;
- A0-A5 assistance attribution on checkpoint evidence;
- daily outcomes: achieved, partially achieved, blocked, and misdirected;
- local SQLite storage under `%LOCALAPPDATA%\Kovacs\v0.3`;
- evidence-based Staff Engineer competency model;
- memory review, confirmation, pin, and delete controls;
- invocation count, prompt-character, latency, and image-use telemetry;
- V0.2 authorized observation and privacy controls;
- explicit End Day evidence form and V0.1 Debrief;
- secure V0.3 Electron renderer and fixed IPC bridge.

## Excluded

- Google Meet captions or Meeting Mode;
- Career Mode, CV workflows, job sites, and social-media workflows;
- direct OpenAI API or Realtime API integration;
- autonomous computer use or post-session actions;
- hidden or undetectable operation;
- raw screenshot, transcript, audio, or browser-content retention;
- cloud synchronization;
- automatic acceptance of inferred or consequential memories;
- productivity scores based on hours, clicks, windows, or model-call volume.

## Release rule

V0.3.0 is complete only when contracts, lifecycle tests, SQLite restart behavior, memory controls, token telemetry, V0/V0.1/V0.2 regressions, build, audit, and a Windows pet smoke session pass.
