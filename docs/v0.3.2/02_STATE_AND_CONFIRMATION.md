# State and confirmation

Natural input never directly changes the operating contract.

## Calibration

`unconfigured -> interpreting -> pending setup proposal -> confirmed profile`

The proposal separates explicit, inferred, unknown, and confirmed values. Confirmation accepts the reviewed interpretation and stores only established values as visible local memories.

## Day

`configured -> natural intent -> pending day proposal -> active day`

The project directory remains an advanced local setting because it scopes Codex CLI and screen authorization. The user-facing input is the desired movement or situation.

## End Day

`active day -> natural debrief -> pending End Day proposal -> confirmed closed day`

Drafting does not stop observation, record evidence, update competency, or retain a lesson. Only confirmation calls the existing deterministic close path. Rejection leaves the day active.
