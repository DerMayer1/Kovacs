# Kovacs V0.3.2 — Context Foundation

V0.3.2 makes Kovacs easier to brief and better at understanding work context without expanding its authority. The user writes naturally; Codex CLI proposes structure; explicit confirmation remains the only transition into durable operating state.

## Included

- One-input initial calibration with sourced interpretations, confidence, assumptions, and at most two consequential questions.
- One-input daily intent transformed into an objective, evidence criteria, and bounded checkpoints.
- One-input End Day transformed into a conservative evidence proposal before the day can close.
- Windows accessibility and OCR adapters invoked locally on an already-authorized capture.
- A compact context engine that persists categories, confidence, ambiguity, changed fields, and a digest—never raw OCR, accessibility text, screenshots, or window titles.
- Deterministic local vector memory and hybrid retrieval with provenance.
- Additive SQLite migration, recovery, retention, backup, and deletion behavior.

## Explicit exclusions

- No direct OpenAI API integration. Model reasoning continues through Codex CLI.
- No autonomous click, typing, sending, submission, file mutation, or post-meeting action.
- No continuous model loop. Existing event and cooldown gates remain authoritative.
- Meeting Mode and Career Mode remain reserved for later releases.

## Success definition

V0.3.2 is releasable when automated validation proves the privacy boundary, local retrieval, confirmation gates, migration, security posture, and V0.3.1 regressions; live acceptance separately proves the real Codex CLI schemas and native Windows perception path.
